PROBLEM & SOLUTION
⚠ PROBLEM
  • web/was 이중화 환경에서 특정 서버로 트래픽이 집중되어 부하 불균형 발생
  • 서버 재기동 시 DRM 서비스가 자동으로 구동되지 않아 수동 개입 필요
  • 내부 대시보드 연동 시 필수 URL 경로가 차단되어 기능 장애
  • 모의해킹 사전 준비 미흡으로 보안 점검 대응 시간 낭비
✓ SOLUTION
  • Nginx server.conf 수정으로 트래픽 균등 분산 — 로드밸런싱 최적화
  • .service 파일 작성 및 Linux 데몬 등록 — 재기동 시 DRM 자동 구동 보장
  • Location 블록 재설정으로 특정 URL 패턴만 허용, 차단 문제 해소
  • 소나큐브 정적분석 + 로컬 환경 사전 세팅으로 모의해킹 선제 대응

CODE SNIPPETS
nginx.conf — upstream 로드밸런싱 설정 Nginx Config 
# ****** 운영 시스템의 실제 설정이 아닌 핵심 로직을 재현한 코드입니다 ******

# ── upstream: WAS 이중화 서버 등록 ─────────────────────────────────────
upstream portal_was {
    # 기본값: Round Robin — 요청을 순서대로 균등 분배
    server 192.168.10.11:8080 weight=1;
    server 192.168.10.12:8080 weight=1;

    # keepalive: upstream 연결 재사용으로 커넥션 오버헤드 감소
    keepalive 32;
}

# ── server 블록: 리버스 프록시 + 보안 라우팅 ─────────────────────────
server {
    listen 443 ssl;
    server_name portal.example.internal;

    # 일반 요청 → WAS 로드밸런싱
    location / {
        proxy_pass         http://portal_was;
        proxy_set_header   Host              $host;
        proxy_set_header   X-Real-IP         $remote_addr;
        proxy_set_header   X-Forwarded-For   $proxy_add_x_forwarded_for;
        proxy_http_version 1.1;
        proxy_set_header   Connection        "";
    }

    # 내부 대시보드 연동 — 특정 URL 패턴만 허용 (보안 라우팅 재설정)
    location ~* ^/dashboard/(api|metrics)/ {
        allow 192.168.0.0/16;   # 내부망만 허용
        deny  all;
        proxy_pass http://portal_was;
    }
}
개선 포인트: 기존 설정에서 특정 WAS 서버로 트래픽이 집중되던 문제를 upstream Round Robin + keepalive 설정으로 해결. 내부 대시보드 연동 시 차단됐던 경로는 Location 블록에서 URL 패턴 기반으로 선별 허용하여 보안과 기능을 동시에 확보했습니다.
drm-agent.service — Linux systemd 데몬 등록 systemd Unit 
# ****** 운영 시스템의 실제 설정이 아닌 핵심 로직을 재현한 코드입니다 ******

# /etc/systemd/system/drm-agent.service
# 서버 재기동 시 DRM 서비스 자동 구동 보장을 위한 systemd unit 파일

[Unit]
Description=DRM Agent Service
# 네트워크 인터페이스가 준비된 후 기동
After=network.target

[Service]
Type=forking
User=drm-user
WorkingDirectory=/opt/drm-agent
ExecStart=/opt/drm-agent/bin/start.sh
ExecStop=/opt/drm-agent/bin/stop.sh

# 비정상 종료 시 자동 재시작 (30초 후)
Restart=on-failure
RestartSec=30

# 표준 출력·에러를 syslog에 남겨 journalctl로 조회 가능
StandardOutput=journal
StandardError=journal

[Install]
# multi-user.target: 일반 서버 runlevel에서 자동 활성화
WantedBy=multi-user.target

# ── 등록 명령어 ──────────────────────────────────────────────────────
# sudo systemctl daemon-reload
# sudo systemctl enable drm-agent   ← 부팅 시 자동 시작 등록
# sudo systemctl start drm-agent
# sudo systemctl status drm-agent   ← 상태 확인
개선 포인트: 기존에는 서버 재기동 후 DRM 서비스를 수동으로 실행해야 했습니다. systemd unit 파일을 작성하고 enable 등록으로 재기동 후 자동 구동을 보장했으며, Restart=on-failure로 비정상 종료 시에도 자동 복구됩니다.
