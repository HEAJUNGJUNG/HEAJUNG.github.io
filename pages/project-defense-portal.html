방산 협력사의 보안 자산 관리 시스템을 풀스택으로 신규 개발하고 운영 배포까지 담당한 프로젝트입니다. Spring Batch 기반 SMS·이메일 자동 알림, POST API 보안 설계, CSV 대량 업로드 파이프라인을 단독으로 설계·구현하였으며 모의해킹 대응 및 Linux 서버 배포 운영을 병행했습니다.

PROBLEM & SOLUTION
⚠ PROBLEM
  • ✗ 방산 협력사 자산 등록·변경 이벤트를 수동으로 확인해야 해 담당자 누락 빈번
  • ✗ 수백 건 자산 데이터를 엑셀 수기 입력으로 처리해 오류·시간 낭비
  • ✗ 비동기 흐름 제어 부재로 화면 블로킹 현상 발생
✓ SOLUTION
  • ✓ Spring Batch로 SMS·이메일 알림 자동 발신 파이프라인 구축
  • ✓ CSV 업로드 → DTO 파싱 → DB 일괄 적재 + 검증 로직으로 안정성 확보
  • ✓ async/await + Ajax로 비동기 흐름 제어 및 UX 개선

// ── BatcbService.java  반납일 지난 자산유무 확인 및 알림메일 발송 서비스 로직 ─────────────────────────────────────────────
@Configuration
@RequiredArgsConstructor
public class NotificationJobConfig {

    private final JobBuilderFactory jobBuilderFactory;
    private final StepBuilderFactory stepBuilderFactory;
    private final AssetNotificationReader notificationReader;
    private final SmsItemWriter smsItemWriter;
    private final EmailItemWriter emailItemWriter;

    // 자산 만료 알림 Job - 스케줄러에 의해 주기적 실행
    @Bean
    public Job assetNotificationJob() {
        return jobBuilderFactory.get("assetNotificationJob")
                .start(smsNotificationStep())
                .next(emailNotificationStep())
                .build();
    }

    // SMS 발송 Step: 청크 단위(50건) 처리
    @Bean
    public Step smsNotificationStep() {
        return stepBuilderFactory.get("smsNotificationStep")
                .<AssetNotificationDto, SmsPayload>chunk(50)
                .reader(notificationReader)
                .processor(smsProcessor())  // 수신자·메시지 포맷 변환
                .writer(smsItemWriter)       // 외부 SMS API 호출
                .build();
    }

    // Email 발송 Step
    @Bean
    public Step emailNotificationStep() {
        return stepBuilderFactory.get("emailNotificationStep")
                .<AssetNotificationDto, EmailPayload>chunk(50)
                .reader(notificationReader)
                .processor(emailProcessor())
                .writer(emailItemWriter)
                .build();
    }
// ── BatchController.java  배치 코드로 서비스를 구분 , 할당된 배치 코드로 해당하는 서비스를 호출 ────────────────────────────────

}

assetUpload.js JavaScript · async/await 
// CSV 파일 → 파싱 → API 전송 비동기 파이프라인
async function handleCsvUpload(file) {
    const uploadBtn = document.getElementById('uploadBtn');

    try {
        uploadBtn.disabled = true;
        showSpinner('CSV 파일을 분석 중입니다...');

        // 1. CSV 파싱 (FileReader → Promise 래핑)
        const rawData = await parseCSV(file);

        // 2. 클라이언트 사전 검증 (빈 행, 필수 컬럼 체크)
        const validated = validateRows(rawData);
        if (validated.errors.length > 0) {
            showValidationErrors(validated.errors);
            return;
        }

        showSpinner('서버에 업로드 중입니다...');

        // 3. POST 전송 (URL 파라미터 없이 RequestBody)
        const response = await fetch('/api/assets/upload', {
            method: 'POST',
            headers: { 'Content-Type': 'application/json',
                        'X-CSRF-TOKEN': getCsrfToken() },
            body: JSON.stringify({ assets: validated.data })
        });

        const result = await response.json();

        if (!response.ok) throw new Error(result.message);

        showSuccess(`✓ ${result.data} 자산이 등록되었습니다.`);
        refreshGrid();  // agGrid 데이터 갱신

    } catch (err) {
        showError('업로드 실패: ' + err.message);
    } finally {
        uploadBtn.disabled = false;
        hideSpinner();
    }
}
UX try/finally 구조로 에러 발생 시에도 버튼 비활성화 해제 보장. 단계별 스피너 메시지로 사용자 피드백 강화. 클라이언트 1차 검증으로 불필요한 서버 요청 방지.
